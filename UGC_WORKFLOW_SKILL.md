# AI短视频 UGC 口播讲解类复刻 — 唯一 Skill 文件

> **小白只看这一份文件就够了。** 其他文件（`workflow_state.yaml`、`.cursor/skills/`）都是可选辅助，不用管也能跑通。

> **给 AI 的说明**：你是「向导」。严格按本文步骤推进。能在对话里完成的直接完成；不能完成的（看视频、即梦出片）只输出「你去哪 + 复制什么 + 做完贴回什么」。每步结束用「📍 进度块」帮用户记录，方便下次续跑。

---

## 一、工作模式（30 秒看懂）

想象有 **4 个角色**，你只需要认识前 2 个：

```
你（老板）  ←→  向导 AI（CodeX / Cursor / 任意聊天，读本文）
                    ↓ 偶尔让你去
              Gemini 网页（只看对标视频）
              即梦网页（做参考图 + Seedance 2.0 出片）
```

**循环就一种**：向导 AI 让你做一件事 → 你做完把结果贴回来 → 向导 AI 做下一件事。

| 谁来做 | 做什么 |
|--------|--------|
| **你** | 找对标视频、去 Gemini/即梦点几下、选满意的成片、发布 |
| **向导 AI** | 改口播稿、写 15 秒脚本、清洗、裂变 10 条、写 TikTok 文案 |
| **Gemini** | 上传对标视频，拆出「口播稿1」和人物画像（AI 看不了视频） |
| **即梦** | 产品图融合成参考图；Seedance 2.0 生成最终视频 |

**全程只需 2 个网站 + 1 个 AI 对话**：Gemini、即梦、你的向导 AI。

---

## 二、唯一启动口令（复制发给你的向导 AI）

在 CodeX / Cursor 里先 `@` 本文件，再粘贴：

```
请按 UGC_WORKFLOW_SKILL.md 担任向导，带我复刻一条带货短视频。

我的情况：
- 产品有文字介绍（没有就写「只有产品图，服饰类」）
- 对标视频是口播类（不是口播就写「非口播对标」）

产品信息：
（粘贴你的标题+五点描述，或写「见附件产品图」）

从第 0 步开始，一步一步带我，每次只说当前这一步该做什么。
```

以后每次续跑，只要说：**「继续 UGC 工作流，进度块如下：」** 并粘贴向导 AI 上次给你的 📍 进度块。

## 三、完整模拟演示（虚构案例，跟着走一遍就懂）

**设定**：你要卖「开放式耳机」。对标是一条 TikTok 口播爆款。产品有亚马逊五点描述。

---

### 第 0 步：你找对标（自己完成，不用 AI）

你在 TikTok 看到一条高播放耳机口播视频，下载保存为 `爆款耳机.mp4`。

---

### 第 1 步：去 Gemini 看视频（向导 AI 不能替你做）

**向导 AI 会说：**

> 请打开 gemini.google.com → 上传 `爆款耳机.mp4` → 先发下面 Prompt A，再发 Prompt B。做完把两段回复完整贴回来。

**你去 Gemini，粘贴 Prompt A**（全文见本文第六节「step1 — Prompt A」）→ 得到 **【爆款模块】** 等拆解。

**再粘贴 Prompt B** → 得到：
- 【人物画像】25岁美国男生，口语快，激动
- 【情绪顺序】惊讶→吐槽→安利→催促
- 【口播稿1】「Wait, you're STILL putting earbuds IN your ears?...」

**你贴回向导 AI：**

> Gemini 结果如下：（粘贴上面全部）

**向导 AI 回复并给你 📍 进度块：**

> 收到。口播稿1 已记录。进入第 2 步，我根据你的产品信息改写口播稿……
>
> 【口播稿2】「Okay so these open-ear buds don't go in your ear canal at all...」
>
> ```
> 📍 进度：第2步完成
> 口播对标：是 | 有产品文案：是
> 爆款模块：前3秒反问+夸张表情
> 人物画像：（略）
> 情绪顺序：（略）
> 口播稿2：（全文）
> ```

---

### 第 2 步：改口播稿（向导 AI 在对话里直接做，你不用跳转）

已在上面模拟中完成。你什么都不用点，看结果是否满意，不满意就说「钩子再狠一点」。

---

### 第 3 步：生成 15 秒脚本（向导 AI 直接做）

**向导 AI 输出 【脚本1】：**

```
A 15-second vertical style ad for open-ear earbuds, energetic,
shot on iPhone, natural daylight, realistic, no brand name

0–3s | Guy in car glances at camera, shocked | Guy says: "You're still jamming buds IN your ears?" — 惊讶/重音在 STILL
3–8s | Shows open-ear bud near ear, doesn't insert | Guy says: "These sit outside—zero ear pressure..." — 吐槽/轻松
8–15s | Walking outside, buds visible | Guy says: "Link's right there if you run hot..." — 催促/指镜头
Voice: mid-20s American male, fast casual
```

并更新 📍 进度块（含脚本1）。

---

### 第 4 步：清洗脚本（向导 AI 直接做）

去掉「入耳」「硅胶」等形态词，输出 **【清洗版脚本】**。若某句钩子不能删，向导会标「⚠️ 需你决定」。

---

### 第 5 步：去即梦做参考图（你手动）

**向导 AI 会说：**

> 打开 jimeng.jianying.com → 图片生成/编辑 → 上传你的产品图 + 对标视频截图 → 粘贴下面这段话 → 保存高清图。

并给你一段即梦用的话（见本文 step_refimage）。

你做完后告诉向导：**「参考图做好了」**（可选发图预览）。

---

### 第 6 步：去即梦 Seedance 2.0 出片（你手动）

**向导 AI 会说：**

> 即梦 → 视频生成 → 选 **Seedance 2.0** → 上传参考图 → 粘贴【清洗版脚本】→ 生成 → 下载视频。

你下载成片，自己看网感。不满意就说「第 3 段语气再急一点」让向导改脚本重来。

---

### 第 7 步：裂变 10 条（向导 AI 直接做）

你说：**「继续，帮我裂变 10 条不同人物场景」**

向导一次给出 10 条变体脚本。你想出哪条，再拿去即梦出片。

---

### 第 8 步：TikTok 文案（向导 AI 直接做）

向导给每条成片 3 条英文文案 + 5 个 tag。

---

### 演示结束 — 你实际每天怎么用

1. 新开对话 → `@UGC_WORKFLOW_SKILL.md` + 启动口令（第二节）
2. 向导让你去外链时就去一下，回来粘贴
3. 对话太长就新开，粘贴上次 **📍 进度块** 即可续跑
4. **不需要 yaml 文件**，进度块在聊天里跟着走就行

### 逐步工作流与 Prompt 原文

### step0 — 找对标（人）

在数据平台找高播放、高转化、有口播的视频（非纯 BGM）。优先 AI 爆款（剧情冲突/口播），其次有口播实拍。找实拍难做的画面：外国人、本土口播、冲突剧情、罕见场景。前期追求量，用数据回调脚本。

完成后：将视频放入 `inputs/`，更新 `workflow_state.yaml` 的 `inputs.ref_video`，`current_step: step1`。

---

### step1 — 拆解对标视频（外链 Gemini）

**操作**：打开 Gemini → 上传对标视频 → **先发 Prompt A**，再发 **Prompt B**。

#### Prompt A（三层拆解）

```
角色

你是一位短视频爆款拆解专家,擅长从转化视角逆向解构 UGC 风格广告视频。

任务

我会直接发给你一条短视频。请你观看后,按以下三层逐项拆解,目标是抽象出可复用的"爆款模块",而非泛泛复述内容。注意结合画面、声音、节奏综合判断,不要只看字幕。

拆解维度

一、内容层(决定起量)

1. 前3秒钩子:分别拆视觉钩子、文案钩子、声音钩子,明确"它用什么让人停下来"。

2. 选题痛点:踩的是显性需求还是隐性情绪(身份/焦虑/爽点)?

3. 脚本结构:按时间轴切段(钩子→痛点放大→方案出现→效果证明→CTA),标出每段时长占比和大致时间码。

4. 情绪曲线:逐段标注情绪走向,定位情绪峰值出现在第几秒。

二、表现层(决定 UGC 感与可信度)

1. 镜头语言:景别、转场、切镜节奏,判断是真手持 UGC 还是伪装精致广告。

2. 产品植入:产品第几秒出现、用什么场景带出、有无前后效果对比。 3. 听觉:BGM 类型、卡点位置、口播语气、关键音效。 4. 字幕文案:字幕样式、句式(短句/疑问/口语化程度)。 

三、转化层(决定变现) 

1. CTA:引导动作的位置、话术、强弱,以及如何把情绪势能导向点击。

2. 数据反推(若我提供):完播/互动/转化分别强在哪,倒推真正的功臣环节。 

输出要求

每个维度先给结论,再给依据,不要平均用力。

全程标注时间码(如 0:00-0:03),让我能对着视频核对。

最后单独输出一段【可复用模块】:用一两句话提炼这条视频最强的 1-2 个杠杆(通常是钩子或情绪峰值),写成我能直接嫁接到自己产品脚本里的抽象公式。

全程口语化、问题导向,不要电影术语,不要冗长解释。

现在开始拆解我发的视频。
```

**预期输出**：【爆款模块】

#### Prompt B（人物画像+口播稿）

```
拆解这个视频：标注口播人物的年龄、口音、音色、情绪起伏、重音；拆解画面元素和点赞观看的原因、情绪变化；给我全部口播稿。
```

**预期输出**：
- 【人物画像：年龄/口音/音色/重音/画面】
- 【情绪顺序】
- 【口播稿1】

**完成后**：写入 `artifacts`，`steps_completed` 加 `step1`，`current_step: step2`。

```

**输出**：【口播稿2】

### step2b — 口播稿转对话稿（仅 `ref_video_type = non_talking`）

SOP 未单独写 prompt，本 Skill 补充桥接步骤：

```
将以下【口播稿2】改写成【人物对话稿2】：
- 保持爆款金句、钩子、结尾、情绪起伏不变
- 改为多人物对话或旁白+对话形式（适合非口播对标视频）
- 不要写字幕、不要品牌名
- 口语化，适合 15s 短视频

【口播稿2】
（粘贴）
```

**输出**：【人物对话稿2】

**完成后**：`current_step: step3`。

### step3 — 生成脚本（AI 对话内执行）

根据 `ref_video_type` 选一支：

#### 分支 A：口播对标 `talking`

```
我要ai生成视频，现在需要生成脚本。
 风格：UGC、iPhone 随手拍、handheld、轻微晃动、自然光、生活原声。禁止 cinematic / film look / color grading / 运镜打光术语。
 输入：
【人物人画像年龄、口音、音色、重音、画面】
【情绪顺序】
【口播稿2】


 要求：

- 输出15s的脚本，节奏快

- 按时间轴分段，每段同时含 画面 + 说话人 + 台词 + 情绪重音

- 说话的人要写进画面（谁在说、是否对镜头），口型模型同步

### step_refimage — 参考图（外链）

**图的作用**：定人物场景风格、保留产品细节质感、定光影调性。

**做法**：自己的产品图 + 对标视频截图 → 融合成一张高清图。图中人脸需在眼鼻部打马赛克。

**即梦 / ChatGPT Image 复制块**：

```
将附件中的「产品图」与「对标视频截图」融合成一张高清竖屏参考图：
- 保留产品细节质感与真实光影
- UGC 随手拍、自然光、生活感
- 如截图有人脸，眼鼻部打马赛克
- 不要文字、不要水印
```

保存路径写入 `artifacts.参考图路径`。

### step6 — 脚本 + 图，Seedance 出片（外链即梦）

1. 打开即梦 → 视频生成 → 选择 **Seedance 2.0**
2. 上传 `参考图路径` 中的高清图（优先实拍图，其次白底图）
3. 粘贴 **【清洗版脚本】** 全文作为生成提示
4. 下载成片到 `runs/{run_id}/outputs/`

**完成后**：更新 `成片路径列表`，`current_step: step7` 或人工选片。
